# «Лидер Плюс» — онлайн-запись на консультацию

Приложение записи к двум консультантам (Сымбат и Айгерим). У каждой своё
расписание и свои записи. Клиент выбирает консультанта → класс → день →
время → оставляет контакты. Кабинет защищён PIN-кодом.

## Что нужно (всё бесплатно)
- Аккаунт **GitHub** — github.com
- Аккаунт **Supabase** (база данных) — supabase.com
- Аккаунт **Vercel** (хостинг + ссылка) — vercel.com
- На компьютере **Node.js LTS** — nodejs.org (нужен только если хотите запускать локально)

────────────────────────────────────────────────────────────

## Шаг 1. База данных (Supabase)
1. supabase.com → войдите через Google → **New project**.
2. Имя `lider-plus`, придумайте и сохраните пароль базы, регион — Frankfurt.
3. Когда проект готов: слева **SQL Editor → New query**.
4. Откройте файл `supabase_setup.sql`, скопируйте всё, вставьте, нажмите **Run**.
   Должно появиться «Success».
5. Слева **Project Settings (шестерёнка) → API**. Скопируйте:
   - **Project URL**
   - ключ **anon public**

## Шаг 2. Загрузить код на GitHub
Вариант без терминала:
1. github.com → **New repository** → имя `lider-plus` → Create.
2. На странице репозитория: **uploading an existing file**.
3. Перетащите СОДЕРЖИМОЕ этой папки (кроме `node_modules` и `dist`).
4. **Commit changes**.

## Шаг 3. Опубликовать (Vercel)
1. vercel.com → **Sign up with GitHub**.
2. **Add New → Project** → выберите репозиторий `lider-plus` → **Import**.
3. Раздел **Environment Variables** — добавьте две переменные:
   | Name                  | Value                          |
   |-----------------------|--------------------------------|
   | `VITE_SUPABASE_URL`   | ваш Project URL из шага 1      |
   | `VITE_SUPABASE_ANON`  | ваш anon public ключ из шага 1 |
4. **Deploy**. Через минуту получите ссылку вида `lider-plus.vercel.app`.
   Это и есть рабочий адрес записи — отправляйте его клиентам.

## Шаг 4. Запуск к клиентам
- Вставьте ссылку в шапку Instagram / WhatsApp центра.
- Сделайте QR этой ссылки (любой генератор QR) и распечатайте в центре.

## Настройки, которые стоит поменять
Файл `src/App.jsx`, в самом верху:
- `ADMIN_PIN` — PIN для входа в кабинет (сейчас `2468`).
- `CONSULTANTS` — имена консультантов. **Поле `id` (symbat/aigerim) после
  запуска не меняйте** — по нему хранятся записи.

После любой правки кода на GitHub Vercel пересоберёт сайт автоматически.

## Как пользуется консультант
Открывает ссылку → вкладка **Кабинет** → вводит PIN → выбирает себя
(Сымбат / Айгерим) → вкладка **Расписание**: отмечает рабочие дни и часы.
Вкладка **Записи** показывает, кто записался; телефон кликабельный, запись
можно отменить крестиком.

## Запуск локально (необязательно)
```
npm install
npm run dev
```
Без ключей Supabase приложение работает в демо-режиме (записи только на
этом устройстве) — удобно показать, но для реальной записи нужны шаги 1–3.
